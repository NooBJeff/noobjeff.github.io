May 27, 2017

为了兴趣爱好，为了提高技术，为了申请大学有目标，为了简历好看。

本套博客记录编写计算机图形学下的实时软件渲染器。

我的编程学习过程谈不上努力刻苦，因此在写代码的过程中肯定会有很多问题，欢迎各位读者到GitHub的本Repo中提出Issue什么的 :)

准备工作

  1. 下载我们用于学习的教材 《Fundamentals of Computer Graphics》 (第三版， pdf 格式，英文原版)

    • 链接指向 Dropbox，请自备翻墙。
    • 很抱歉这是英文版的，因为我找不到中文版啊啊啊。如果各位谁有中文版的话，一定要贡献出来啊！英文看的咪咪痛
    • 本文件仅用于学习目的，请于下载后24小时内自行删除Blah Blah Blah

  2. 下载 Visual Studio 2017 。写渲染器这种大型一点的项目，不用IDE会比较难受。

  3. 获取我的Repo或者用 VS2017 创建 C++ 的 Win32 项目（不是Win32 控制台应用程序）

项目简介

我的目标是在暑假结束前写出简单的，具有基本明暗效果的渲染器，可以用来查看.obj格式的三维模型。这个软件基本的功能为载入模型，围绕模型移动镜头。其他任何功能都先不管，只注重渲染这一部分。

下面是具体一点的参数

  • 操作系统：Windows（因为需要创建GUI而又不想去学跨平台的GUI库，所以就只能Window“独占”了）
  • 编程语言：C++ （然而由于本人的编程水平，恐怕会写成C with Class）
  • GUI库：Win32 API（最底层的那一套，其实应该用Qt的，但是目前不想去学那个）
  • 线性代数库：由于渲染过程涉及到大量的数学计算，因此需要一个好的数学库。不建议自己造轮子，性能什么的肯定不行。具体用什么第三方库还没找好。

和普通的控制台程序看起来是不是有点不太一样？不要慌，代码中那些参数具体是什么意思，我也不懂，反正也用不上(￣、￣)

有兴趣查看他们的具体意思的话，可以去MSDN搜索。

由于通过Windows API与操作系统进行交互并非本程序的主要目的，所以完全没有必要浪费太多时间学习这些既底层，又不实用的API。编写过程中需要使用的话，我都会说明一下的。

如果使用的是较老的VS，那么文件的默认编码一般不是UTF-8。如果代码中有中文处理起来会比较费事，建议在菜单栏的 文件->高级保存选项中选择UTF-8不带签名格式保存。而新版的VS 2017似乎会在检测到Unicode字符后提示用户用UTF-8保存，因此应该不用手动设置文件格式。
